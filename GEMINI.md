# GEMINI.md

# Project Overview
This project is a social media web application that allows users to create accounts, manage personal profiles, and interact with posts created by themselves and other users.

# Core Goal
The goal of this project is to build a functional social platform where authenticated users can publish content, view posts from others, and manage their own posts through full CRUD operations.
Clean minimalistic colrs and design for the frontend.

# Key Features

1. User Authentication
- Users can sign up with an account.
- Users can sign in securely.
- Only authenticated users can access core features.

2. User Profiles
- Each user has a personal profile.
- Profiles store user-specific information and identity within the platform.

3. Post Management (CRUD)
- Create posts
- Read posts
- Update posts
- Delete posts

4. Feed Interaction
- Users can view posts created by other users.
- Posts are displayed in a shared feed or listing.

# System Rules
- Authentication must be verified before allowing post creation or profile interaction.
- Users can only edit or delete posts that they created.
- All users can read posts from other users.

# Purpose of the Project
This project is designed to practice and demonstrate backend and full-stack development concepts including:
- User authentication
- Data persistence
- CRUD operations
- API or server-side logic
- User-content relationships

# Expected Outcome
A clean, stable, and functional social media platform where users can:
- Register and log in
- Create and manage personal profiles
- Create, edit, delete, and view posts
- View posts from other users

The AI should prioritize stability, clarity, maintainability, and completion of the core platform features.
